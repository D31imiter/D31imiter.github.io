#### 三种邮箱协议：

```
SMTP,Simple Mail Transfer Protocol，简单邮件传输协议，25端口
1. 基于文本
2. 通过推送实现，无法实现拉取
3. 属于TCP/IP协议簇，建立在FTP文件传输服务之上的一种邮件服务
工作过程：
1. 建立连接：SMTP客户端请求与SMTP服务器建立一个TCP连接，连接成功后互通域名等一系列相关信息
2. 邮件发送：SMTP客户将邮件发送给SMTP服务器，SMTP服务器进行接收
3. 连接释放：SMTP退出命令，服务器处理相应，随后关闭TCP连接

POP，Post Office Protocol，邮局协议，110端口，常用第三版协议，简称POP3
1. cs架构
2. 默认使用TCP/IP协议进行传输
3. 在TCP/IP协议簇中属于应用层协议
4. POP3协议访问模式为离线模式，即需要将邮件下载至客户用户端才能进行查看，且在客户端上进行如删除邮件之类的操作服务器是无法接收到客户端的信息
工作过程：
1. Client通过SMTP将邮件发送到SMTP服务器
2. SMTP服务器通过互联网将邮件发送到对方POP所在的服务器
3. pop客户端可以选择是否下载邮件（该操作是离线操作，即服务端口和其他用户不可知）

IMAP，Internet Message Access Protocol，交互邮件访问协议，143端口，常用版本为IMAP4
1. 默认传输协议TCP/IP，，应用层协议。分布式，cs架构
2. 支持离线/在线访问模式，此项是POP协议最大的区别
3. 能够持续不间断的连接服务器，支持服务器保留信息状态。客户端操作服务器端可以看见，支持多个用户同时访问且能让用户感知其他用户的操作
4. 支持访问消息中的MIME部分和部分获取
工作过程：同POP3，区别在于最后一步，不一定发送，而是用户直接在服务端上进行操作

三者区别：
1. SMTP是明确的在TCP/IP协议簇里，其他两者只是通过TCP/IP协议进行传输
2. POP与IMAP主要区别就在于POP是单一的离线模式对邮件进行传输，且pop与服务端笑消息不互通
3. SMTP是中继推送，无拉取功能
```
